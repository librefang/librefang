# LibreFang - Agent Operating System

LibreFang 是一个用 Rust 编写的开源 Agent 操作系统，是 OpenFang 的社区维护分支。

<Note>
- **官网**: [https://librefang.ai/](https://librefang.ai/)
- **GitHub**: [https://github.com/librefang/librefang](https://github.com/librefang/librefang)
- **特性**: 15 个 Rust crates，241K+ 代码行，16 层安全防护，15 个 Hands
</Note>

## 核心概念

### Hands - 自主工作的 Agent

Hands 是 LibreFang 的核心创新——预构建的自主能力包，独立运行、按计划执行，无需人工干预。

| Hand | 功能 |
|------|------|

| [**API Tester**](https://github.com/librefang/librefang-registry/tree/main/hands/apitester) | API 端点发现、请求验证、负载测试、回归检测 |
| [**Browser**](https://github.com/librefang/librefang-registry/tree/main/hands/browser) | Web 自动化、填写表单、点击按钮、多步骤工作流 |

| [**Creator**](https://github.com/librefang/librefang-registry/tree/main/hands/creator) | AI 媒体工作室、图像/视频/音乐/语音生成 |
| [**DevOps**](https://github.com/librefang/librefang-registry/tree/main/hands/devops) | CI/CD 管理、基础设施监控、部署自动化、事件响应 |
| [**Lead**](https://github.com/librefang/librefang-registry/tree/main/hands/lead) | 潜在客户发现、信息丰富、评分排序、定时投递 |

| [**Reddit**](https://github.com/librefang/librefang-registry/tree/main/hands/reddit) | 子版块监控、内容发布、回复管理、互动追踪 |

## 安装

### CLI 安装

CLI 是 LibreFang 的核心命令行工具，用于管理 Agent、Hands、工作流等。

**一键安装脚本（推荐）**

Linux / macOS / WSL:
```bash
curl -fsSL https://librefang.ai/install.sh | sh
```

Windows (PowerShell):
```powershell
irm https://librefang.ai/install.ps1 | iex
```

脚本自动检测平台和架构，下载二进制，验证 SHA256 校验和，配置 PATH 并启动守护进程。

环境变量：
- `LIBREFANG_INSTALL_DIR` — 安装目录（默认 `~/.librefang/bin`）
- `LIBREFANG_VERSION` — 指定版本（默认最新）
- `LIBREFANG_AUTO_START` — 是否自动启动（默认 1）

**Homebrew**

```bash
brew tap librefang/tap
brew install librefang              # Stable
brew install librefang-beta         # Beta
brew install librefang-rc           # RC
```

**npm**

```bash
npm install -g @librefang/cli                  # Stable
npm install -g @librefang/cli@next             # 最新预发布版 (Beta 或 RC)
npm install -g @librefang/cli@2026.3.25-rc1    # 指定版本
```

**Cargo**

```bash
cargo install --git https://github.com/librefang/librefang librefang-cli
```

或从源码构建：
```bash
git clone https://github.com/librefang/librefang.git
cd librefang
cargo install --path crates/librefang-cli
```

**Docker**

```bash
docker run -d \
  --name librefang \
  -p 4545:4545 \
  -e ANTHROPIC_API_KEY=$ANTHROPIC_API_KEY \
  -v librefang-data:/data \
  ghcr.io/librefang/librefang:latest
```

Docker Compose：
```bash
git clone https://github.com/librefang/librefang.git
cd librefang/deploy
docker compose up -d
```

**验证安装**

```bash
librefang --version
```

## 快速开始

### 1. 初始化

```bash
librefang init
```

创建 `~/.librefang/` 目录和默认配置文件（可通过 `LIBREFANG_HOME` 环境变量自定义路径）。详见 [CLI 参考](/zh/cli)。

初始化后的目录结构：

```
~/.librefang/
├── config.toml                # 主配置文件
├── aliases.toml               # 模型和命令别名
├── data/
│   ├── librefang.db           #   SQLite 数据库（内存、会话、知识图谱）
│   └── state.db               #   守护进程状态持久化
├── registry/                  # 从 librefang-registry 同步
│   ├── agents/                #   32 个 Agent 模板
│   ├── hands/                 #   15 个 Hand 定义
│   ├── providers/             #   49 个提供商配置
│   ├── skills/                #   技能定义
│   ├── workflows/             #   工作流模板
│   └── plugins/               #   插件定义
├── workspaces/                # 运行时工作目录
│   ├── agents/                #   活跃 Agent 工作目录
│   └── hands/                 #   活跃 Hand 工作目录
├── skills/                    # 用户安装的技能
├── plugins/                   # 用户安装的插件
├── vault.enc                  # 加密凭证金库
├── logs/                      # 应用日志
├── cache/                     # 临时缓存
└── bin/                       # CLI 二进制（脚本安装时）
```

完整目录说明详见 [系统架构 - 用户数据目录](/zh/architecture#用户数据目录)。

### 2. 配置 API Key

LibreFang 需要至少一个 LLM 提供商 API Key。支持 49 个驱动，详见 [LLM 提供商](/zh/providers)。

```bash
# Anthropic (Claude)
export ANTHROPIC_API_KEY=sk-ant-...

# 或 OpenAI
export OPENAI_API_KEY=sk-...

# 或 Groq (有免费额度)
export GROQ_API_KEY=gsk_...
```

也可以通过 CLI 安全存储：`librefang config set-key groq <your-key>`

### 3. 编辑配置

编辑 `~/.librefang/config.toml`，完整配置项参考 [配置文件](/zh/configuration)：

```toml
[default_model]
provider = "groq"
model = "llama-3.3-70b-versatile"
api_key_env = "GROQ_API_KEY"

[network]
listen_addr = "127.0.0.1:4545"
```

### 4. 启动守护进程

```bash
librefang start
```

启动后访问 http://127.0.0.1:4545/ 查看仪表板。守护进程提供 REST API、WebSocket、SSE 和 Web UI，详见 [API 参考](/zh/api)。

### 5. 创建 Agent

从 32 个模板中选择一个创建 Agent，详见 [Agent 模板](/zh/agents)：

```bash
# 创建 hello-world agent
librefang agent spawn agents/hello-world/agent.toml

# 与 agent 聊天
librefang agent chat <agent-id>

# 或使用快捷方式
librefang chat
```

### 6. 激活 Hand

Hands 是自主执行的能力包，详见 [自主 Hands](/zh/hands)：

```bash
# 激活 Researcher Hand
librefang hand activate researcher

# 查看状态
librefang hand status researcher
```

### 7. 连接通道

将 Agent 接入消息平台，支持 44 个通道适配器，详见 [通道适配器](/zh/channels)：

```bash
# 设置 Telegram 通道
librefang channel setup telegram

# 查看所有通道状态
librefang channel list
```

### 8. 安装技能

从 FangHub 安装技能扩展 Agent 能力，详见 [技能开发](/zh/skills)：

```bash
# 搜索技能
librefang skill search "web scraping"

# 安装技能
librefang skill install <source>
```

### 9. 配置工作流

将多个 Agent 编排为 DAG 工作流，详见 [工作流](/zh/workflows)：

```bash
# 创建工作流
librefang workflow create workflow.json

# 运行工作流
librefang workflow run <id> <input>
```

### 10. 运行诊断

```bash
librefang doctor
```

遇到问题参考 [故障排除](/zh/troubleshooting)，生产部署参考 [生产部署](/zh/production)。

### 创建自定义 Agent

创建 `my-agent.toml`：

```toml
name = "my-assistant"
version = "0.1.0"
description = "A helpful assistant"
author = "you"
module = "builtin:chat"

[model]
provider = "groq"
model = "llama-3.3-70b-versatile"

[capabilities]
tools = ["file_read", "file_list", "web_fetch"]
memory_read = ["*"]
memory_write = ["self.*"]
```

然后生成：

```bash
librefang agent spawn my-agent.toml
```

### 内置工具

- `file_read` - 读取文件
- `file_list` - 列出目录
- `web_fetch` - 获取网页
- `web_search` - 搜索网页
- `memory_store` - 存储记忆
- `memory_recall` - 回忆记忆
- `shell` - 执行 shell 命令
- `browser` - 浏览器自动化

## 16 层安全系统

LibreFang 采用深度防御策略，包含 16 个独立的安全层。详见 [安全](/zh/security)。

1. **WASM 双计量沙箱** - 燃料计量 + 时代中断
2. **Merkle 哈希链审计追踪** - 加密链接的完整操作链
3. **信息流污点追踪** - 敏感信息从源到汇的追踪
4. **Ed25519 签名 Agent 清单** - 加密签名的身份和能力集
5. **SSRF 保护** - 阻止私有 IP 和云元数据端点
6. **Secret 零化** - 自动清除内存中的 API 密钥
7. **OFP 双向认证** - HMAC-SHA256 验证
8. **能力门** - 基于角色的访问控制
9. **安全头** - CSP、X-Frame-Options、HSTS
10. **健康端点重写** - 公开/私有健康检查分离
11. **子进程沙箱** - 环境清理 + 进程树隔离
12. **提示注入扫描器** - 检测覆盖尝试
13. **循环守卫** - SHA256 循环检测
14. **会话修复** - 7 阶段消息历史验证
15. **路径遍历防护** - 规范化和符号链接转义防止
16. **GCRA 速率限制器** - 成本感知的令牌桶
