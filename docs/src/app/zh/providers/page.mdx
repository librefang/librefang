# LLM 提供商指南

LibreFang 内置了完整的模型目录，涵盖 **3 种原生 LLM 驱动**、**49 个提供商**、**230+ 个内置模型**和 **23 个别名**。所有提供商均使用以下三种经过充分验证的驱动之一：原生 **Anthropic** 驱动、原生 **Gemini** 驱动，或通用的 **OpenAI 兼容**驱动。本指南是在 LibreFang 中配置、选择和管理 LLM 提供商的唯一权威参考。

模型目录还支持**动态加载** — 官方提供商定义在首次启动时从注册表预安装到 `~/.librefang/providers/`。你可以在同一目录下放置自定义 TOML 文件来添加或覆盖提供商。所有 `~/.librefang/providers/*.toml` 文件在启动时会被合并到目录中，方便你添加私有端点、本地部署或新提供商，无需修改核心配置。

## 快速设置

从零开始最快的方式：

```bash
# 选择一个提供商 - 设置对应的环境变量即可
export GEMINI_API_KEY="your-key"        # 有免费额度
# 或
export GROQ_API_KEY="your-key"          # 有免费额度
# 或
export ANTHROPIC_API_KEY="your-key"
# 或
export OPENAI_API_KEY="your-key"
```

LibreFang 在启动时自动检测哪些提供商已配置 API 密钥。已认证的提供商下的所有模型会立即可用。本地提供商（Ollama、vLLM、LM Studio）无需任何密钥。

对于 Gemini，`GEMINI_API_KEY` 和 `GOOGLE_API_KEY` 均可使用。

## 动态提供商加载

官方提供商定义从注册表预安装到 `~/.librefang/providers/`。要添加或覆盖提供商，请在同一目录下放置自定义 `.toml` 文件。每个文件定义一个提供商：

```toml
# ~/.librefang/providers/my-endpoint.toml
id = "my-endpoint"
display_name = "My Private Endpoint"
driver = "openai_compatible"
base_url = "https://llm.internal.company.com/v1"
api_key_env = "MY_ENDPOINT_KEY"
key_required = true

[[models]]
id = "my-model-7b"
display_name = "My Model 7B"
tier = "Balanced"
context_window = 32768
max_output_tokens = 4096
input_cost_per_m = 0.0
output_cost_per_m = 0.0
supports_tools = true
supports_vision = false
```

`~/.librefang/providers/` 中的文件在启动时加载，并与内置提供商一起合并到目录中。

### 提供商区域

部分提供商提供区域特定的端点。区域在注册表 TOML 文件中定义，并支持可选的 `api_key_env` 覆盖：

```toml
# 在提供商的注册表 TOML 中：
[provider.regions.intl]
base_url = "https://dashscope-intl.aliyuncs.com/compatible-mode/v1"

[provider.regions.china]
base_url = "https://api.minimaxi.com/v1"
api_key_env = "MINIMAX_CN_API_KEY"    # 可选：覆盖默认的 API 密钥环境变量
```

在 `config.toml` 中选择区域：

```toml
[provider_regions]
qwen = "intl"
minimax = "china"
```

**优先级：** 区域选择在显式的 `[provider_urls]` 条目*之前*应用。如果同一提供商同时设置了两者，`provider_urls` 优先。

## 每个 Agent 的模型覆盖

`config.toml` 中的每个 Agent 都可以指定自己的模型，覆盖全局默认值：

```toml
# 全局默认模型
[agents.defaults]
model = "claude-sonnet-4-20250514"

# 逐 Agent 覆盖：使用别名或完整模型 ID
[[agents]]
name = "orchestrator"
model = "opus"                      # claude-opus-4-20250514 的别名

[[agents]]
name = "ops"
model = "llama-3.3-70b-versatile"   # 便宜的 Groq 模型，用于简单运维

[[agents]]
name = "coder"
model = "gemini-2.5-flash"          # 快速 + 便宜 + 100 万上下文

[[agents]]
name = "researcher"
model = "sonar-pro"                 # Perplexity，内置网页搜索

# 也可以在 agent manifest TOML 中固定模型
[[agents]]
name = "production-bot"
pinned_model = "claude-sonnet-4-20250514"  # 永不自动路由
```

当 Agent manifest 设置了 `pinned_model` 时，该 Agent 始终使用指定模型，忽略路由配置。此功能用于**稳定模式**（`KernelMode::Stable`），在该模式下模型被冻结以确保生产可靠性。

## 模型路由

LibreFang 可以自动为每个查询选择能够处理该查询的最便宜模型。这通过 `ModelRoutingConfig` 在逐 Agent 级别配置。

### 工作原理

1. **ModelRouter** 根据启发式规则对每个传入的 `CompletionRequest` 进行评分
2. 评分映射到一个 **TaskComplexity** 层级：`Simple`、`Medium` 或 `Complex`
3. 每个层级有一个预配置的模型

### 评分启发式规则

| 信号 | 权重 | 逻辑 |
|--------|--------|-------|
| 总消息长度 | 每 ~4 个字符 1 分 | 粗略的 token 近似 |
| 工具可用性 | 每个定义的工具 +20 | 有工具意味着多步骤工作 |
| 代码标记 | 每个标记 +30 | 反引号、`fn`、`def`、`class`、`import`、`function`、`async`、`await`、`struct`、`impl`、`return` |
| 对话深度 | 每条超过 10 条的消息 +15 | 深层上下文 = 更难的推理 |
| 系统提示长度 | 每 10 个字符（超过 500）+1 | 长系统提示意味着复杂任务 |

### 阈值

| 复杂度 | 分数范围 | 默认模型 |
|-----------|-------------|---------------|
| Simple | `score < 100` | `claude-haiku-4-5-20251001` |
| Medium | `100 <= score < 500` | `claude-sonnet-4-20250514` |
| Complex | `score >= 500` | `claude-sonnet-4-20250514` |

### 配置

```toml
# 在 agent manifest 或 config.toml 中
[routing]
simple_model = "claude-haiku-4-5-20251001"
medium_model = "gemini-2.5-flash"
complex_model = "claude-sonnet-4-20250514"
simple_threshold = 100
complex_threshold = 500
```

路由器还与模型目录集成：
- **`validate_models()`** 检查所有配置的模型 ID 是否存在于目录中
- **`resolve_aliases()`** 将别名展开为规范 ID（例如 `"sonnet"` 变为 `"claude-sonnet-4-20250514"`）


## 后备提供商

`FallbackDriver` 将多个 LLM 驱动包装成一个链。如果主驱动失败，会自动尝试链中的下一个驱动。

### 行为

- 成功时：立即返回
- **速率限制/过载**错误（`429`、`529`）：向上冒泡以进行重试逻辑（不进行故障转移，因为主驱动应在退避后重试）
- **所有其他错误**：记录警告并尝试链中的下一个驱动
- 所有驱动都失败时：返回最后一个错误

### 配置

后备链在 Agent manifest 或 `config.toml` 中配置。当 Agent 处于**稳定模式**（`KernelMode::Stable`）或为提高可靠性配置了多个提供商时，`FallbackDriver` 会自动使用。

```toml
# 示例：主用 Anthropic，后备 Gemini，然后 Groq
[[agents]]
name = "production-bot"
model = "claude-sonnet-4-20250514"
fallback_models = ["gemini-2.5-flash", "llama-3.3-70b-versatile"]
```

后备驱动创建的链为：`AnthropicDriver -> GeminiDriver -> OpenAIDriver(Groq)`。

## 安全说明

- 所有 API 密钥存储为 `Zeroizing<String>` -- 当值从内存中丢弃时，密钥内容会被自动用零覆盖。
- 认证检测（`detect_auth()`）仅检查 `std::env::var()` 是否存在 -- 不会读取或记录实际的密钥值。
- 通过 REST API 设置的提供商 API 密钥（`POST /api/providers/{name}/key`）遵循相同的零化策略。
- 健康检查端点（`/api/health`）不会暴露提供商认证状态或 API 密钥。详细信息位于 `/api/health/detail`，需要认证才能访问。
- 所有 `DriverConfig` 和 `KernelConfig` 结构体的 `Debug` 实现均包含密钥脱敏 -- API 密钥在日志中显示为 `"***"`。
