# Agent 模板目录

LibreFang 提供 **32 个 Agent 模板**，来自 [registry](https://github.com/librefang/librefang-registry/tree/main/agents)。`assistant` 模板已预装，其他模板可从 Dashboard 安装。每个模板都是一个可直接生成的 `agent.toml` 清单文件，位于 `agents/` 目录下。模板覆盖软件工程、业务运营、个人效率和日常任务等领域。

## 快速开始

通过 CLI 生成任意模板：

```bash
librefang spawn orchestrator
librefang spawn coder
librefang spawn --template agents/writer/agent.toml
```

通过 REST API 生成：

```bash
# 通过内置模板名称生成
curl -X POST http://localhost:4545/api/agents \
  -H "Content-Type: application/json" \
  -d '{"template": "coder"}'

# 生成时覆盖参数
curl -X POST http://localhost:4545/api/agents \
  -H "Content-Type: application/json" \
  -d '{"template": "writer", "model": "gemini-2.5-flash"}'
```

向运行中的 Agent 发送消息：

```bash
curl -X POST http://localhost:4545/api/agents/{id}/message \
  -H "Content-Type: application/json" \
  -d '{"content": "Write unit tests for the auth module"}'
```

## 模板层级

模板按任务复杂度和使用的 LLM 模型分为 4 个层级。层级越高，使用的模型能力越强（成本也越高），适用于需要深度推理的任务。

### 第 1 层 -- 前沿 (DeepSeek)

适用于需要最深层推理的任务：多 Agent 编排、系统架构设计和安全分析。

| 模板 | 供应商 | 模型 |
|------|--------|------|
| orchestrator | deepseek | deepseek-chat |
| architect | deepseek | deepseek-chat |
| security-auditor | deepseek | deepseek-chat |

如果 DeepSeek API 密钥不可用，所有第 1 层 Agent 将回退到 `groq/llama-3.3-70b-versatile`。

## 模板目录

### orchestrator

**第 1 层 -- 前沿** | `deepseek/deepseek-chat` | 回退：`groq/llama-3.3-70b-versatile`

> 元 Agent，负责分解复杂任务、委派给专业 Agent 并综合结果。

orchestrator 是 Agent 集群的指挥中心。它分析用户请求，将其拆分为子任务，使用 `agent_list` 发现可用的专业 Agent，通过 `agent_send` 委派工作，在需要时生成新 Agent，并将所有响应综合为连贯的最终答案。它会在执行前说明委派策略，并避免将过于简单的任务进行委派。

- **标签**：无
- **Temperature**：0.3
- **最大 token**：8192
- **Token 配额**：500,000/小时
- **调度方式**：持续检查，每 120 秒一次
- **工具**：`agent_send`、`agent_spawn`、`agent_list`、`agent_kill`、`memory_store`、`memory_recall`、`file_read`、`file_write`
- **能力**：`agent_spawn = true`、`agent_message = ["*"]`、`memory_read = ["*"]`、`memory_write = ["*"]`

```bash
librefang spawn orchestrator
# "Plan and execute a full security audit of the codebase"
```

## 自定义模板

`agents/custom/` 目录保留给你自己的 Agent 模板。按照以下清单格式创建新的 `agent.toml` 文件。

### 清单格式

```toml
# 必填字段
name = "my-agent"
version = "0.1.0"
description = "What this agent does in one sentence."
author = "your-name"
module = "builtin:chat"

# 可选元数据
tags = ["tag1", "tag2"]

# 模型配置（必填）
[model]
provider = "gemini"                  # 供应商：gemini、deepseek、groq、openai、anthropic 等
model = "gemini-2.5-flash"           # 模型标识符
api_key_env = "GEMINI_API_KEY"       # 存放 API 密钥的环境变量
max_tokens = 4096                    # 每次响应的最大输出 token 数
temperature = 0.3                    # 创造性（0.0 = 确定性，1.0 = 创造性）
system_prompt = """Your agent's personality, capabilities, and instructions go here.
Be specific about what the agent should and should not do."""

# 可选回退模型（当主模型不可用时使用）
[[fallback_models]]
provider = "groq"
model = "llama-3.3-70b-versatile"
api_key_env = "GROQ_API_KEY"

# 可选调度（用于自主/后台 Agent）
[schedule]
periodic = { cron = "every 5m" }                                     # 定期执行
# continuous = { check_interval_secs = 120 }                         # 持续循环
# proactive = { conditions = ["event:agent_spawned"] }               # 事件触发

# 资源限制
[resources]
max_llm_tokens_per_hour = 150000    # 每小时 Token 预算
max_concurrent_tools = 5            # 最大并行工具执行数

# 能力授权（最小权限原则）
[capabilities]
tools = ["file_read", "file_write", "file_list", "shell_exec",
         "memory_store", "memory_recall", "web_fetch",
         "agent_send", "agent_list", "agent_spawn", "agent_kill"]
network = ["*"]                     # 网络访问模式
memory_read = ["*"]                 # Agent 可读取的记忆命名空间
memory_write = ["self.*"]           # Agent 可写入的记忆命名空间
agent_spawn = true                  # 此 Agent 是否可以生成其他 Agent？
agent_message = ["*"]               # 可以向哪些 Agent 发送消息？
shell = ["python *", "cargo *"]     # 允许的 Shell 命令模式（白名单）
```

### 可用工具

| 工具 | 描述 |
|------|------|
| `file_read` | 读取文件内容 |
| `file_write` | 写入/创建文件 |
| `file_list` | 列出目录内容 |
| `shell_exec` | 执行 Shell 命令（受 `shell` 白名单限制） |
| `memory_store` | 将键值数据持久化到记忆中 |
| `memory_recall` | 从记忆中检索数据 |
| `web_fetch` | 从 URL 获取内容（具有 SSRF 防护） |
| `agent_send` | 向另一个 Agent 发送消息 |
| `agent_list` | 列出所有运行中的 Agent |
| `agent_spawn` | 生成一个新 Agent |
| `agent_kill` | 终止一个运行中的 Agent |

### 自定义 Agent 提示

1. **从最小配置开始**。仅授予 Agent 实际需要的工具和能力。之后随时可以添加更多。
2. **编写清晰的系统提示**。系统提示是模板中最重要的部分。明确说明 Agent 的角色、方法论、输出格式和限制。
3. **设置合适的 temperature**。精确/分析类任务使用 0.2，均衡任务使用 0.5，创意类任务使用 0.7 及以上。
4. **使用 Shell 白名单**。永远不要授予 `shell = ["*"]`。白名单指定具体的命令模式，如 `shell = ["python *", "cargo test *"]`。
5. **设置 Token 预算**。使用 `max_llm_tokens_per_hour` 防止成本失控。从 100,000 开始，根据使用情况调整。
6. **添加回退模型**。如果主模型有速率限制或可用性问题，添加一个 `[[fallback_models]]` 条目。
7. **使用记忆实现连续性**。授予 `memory_store` 和 `memory_recall` 以便 Agent 可以跨会话持久化上下文。

## 生成 Agent

### CLI

```bash
# 通过模板名称生成
librefang spawn coder

# 生成时指定自定义名称
librefang spawn coder --name "backend-coder"

# 从 TOML 文件路径生成
librefang spawn --template agents/custom/my-agent.toml

# 列出运行中的 Agent
librefang agents

# 发送消息
librefang message <agent-id> "Write a function to parse TOML files"

# 终止 Agent
librefang kill <agent-id>
```

### REST API

```bash
# 从模板生成
POST /api/agents
{"template": "coder"}

# 生成时覆盖参数
POST /api/agents
{"template": "coder", "name": "backend-coder", "model": "deepseek-chat"}

# 发送消息
POST /api/agents/{id}/message
{"content": "Implement the auth module"}

# WebSocket（流式传输）
WS /api/agents/{id}/ws

# 列出 Agent
GET /api/agents

# 删除 Agent
DELETE /api/agents/{id}
```

### OpenAI 兼容 API

```bash
# 通过 OpenAI 兼容端点使用任意 Agent
POST /v1/chat/completions
{
  "model": "librefang:coder",
  "messages": [{"role": "user", "content": "Write a Rust HTTP server"}],
  "stream": true
}

# 列出可用模型
GET /v1/models
```

### Orchestrator 委派

orchestrator Agent 可以通过编程方式生成和委派给任何其他 Agent：

```
User: "Build a REST API with tests and documentation"

Orchestrator:
1. agent_send(coder, "Implement the REST API endpoints")
2. agent_send(test-engineer, "Write integration tests for these endpoints")
3. agent_send(doc-writer, "Document the API endpoints")
4. Synthesize all results into a final report
```

## 环境变量

设置以下 API 密钥以启用对应的模型供应商：

| 变量 | 供应商 | 使用者 |
|------|--------|--------|
| `DEEPSEEK_API_KEY` | DeepSeek | 第 1 层（orchestrator、architect、security-auditor） |
| `GEMINI_API_KEY` | Google Gemini | 第 2 层主模型，第 3 层回退 |
| `GROQ_API_KEY` | Groq | 第 3 层主模型，第 1/2 层回退，第 4 层 |

至少设置 `GROQ_API_KEY` 以启用所有第 3 层和第 4 层 Agent。添加 `GEMINI_API_KEY` 以启用第 2 层 Agent。添加 `DEEPSEEK_API_KEY` 以启用第 1 层前沿 Agent。
