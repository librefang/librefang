# `go-env-greet` — Go (TinyGo) + wit-bindgen-go, exercises `env`

A minimal librefang:plugin Component written in Go. Reads the
`GREETING_NAME` env var via the `librefang:plugin/env` host import
and returns successfully.

## Build

The committed `pre-built/plugin.wasm` is the canonical artefact. To
regenerate it (e.g. after updating the WIT contract):

```bash
cargo xtask plugins-rebuild go-env-greet
```

Under the hood this invokes a three-step pipeline:

```bash
cd examples/plugins/go-env-greet

# 1. Compile to WASI P1 core module
tinygo build -target wasip1 -o /tmp/go-env-greet-core.wasm .

# 2. Embed WIT type information
wasm-tools component embed --world plugin \
    ../../../crates/librefang-skills/wit \
    /tmp/go-env-greet-core.wasm \
    -o /tmp/go-env-greet-embedded.wasm

# 3. Lift to Component Model (WASI P1 adapter bridges TinyGo runtime calls)
wasm-tools component new /tmp/go-env-greet-embedded.wasm \
    --adapt wasi_snapshot_preview1=wasi-adapter/wasi_snapshot_preview1.reactor.wasm \
    -o pre-built/plugin.wasm
```

Requirements: `tinygo` (≥ 0.41), `wasm-tools`, TINYGOROOT set.
The Phase-6 size budget for Go is 2 MB.

## How the source maps to the librefang:plugin world

Go bindings are generated by `wit-bindgen-go` into `librefang/`:
- `librefang/plugin/env/` — `Read(name string)` function
- `librefang/plugin/plugin/` — `Exports.Run` function pointer

`main.go` sets `plugin.Exports.Run = run` and calls `envimport.Read`.
`cabi.go` provides the `cabi_realloc` export required by the Component
Model canonical ABI for string arguments.

The bundled `wasi-adapter/wasi_snapshot_preview1.reactor.wasm` (from
wasmtime v45) adapts TinyGo's WASI P1 runtime calls; the
`librefang:plugin/env` import passes through to the host unchanged.
