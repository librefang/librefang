# Code of Conduct

LibreFang is a community-maintained project. We expect contributors, maintainers, and users to act in good faith and keep technical disagreement constructive.

## Expected Behavior

- Be respectful and direct.
- Critique ideas, code, and decisions without attacking people.
- Give credit when building on someone else's work.
- Keep reviews actionable. If you reject a change, explain why.
- Use public threads for normal project decisions so contributors can see how decisions are made.

## Unacceptable Behavior

- Harassment, threats, hate speech, or personal attacks.
- Dismissing contributors without technical explanation.
- Claiming authorship over someone else's work without attribution.
- Publishing private security details before maintainers have time to respond.
- Retaliation against people who raise concerns in good faith.

## Enforcement

Maintainers may edit, hide, lock, or remove content that violates this policy. Repeated or severe violations can result in temporary or permanent bans from project spaces.

## Reporting

- For normal conduct issues, open an issue in `librefang/librefang` and label it `conduct` if safe to do so.
- For sensitive or security-related problems, use the private reporting path in [`SECURITY.md`](SECURITY.md).
